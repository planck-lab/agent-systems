Runbook: Wenn etwas schiefgehtRunbook: When Things Go Wrong

Runbook-TemplateRunbook Template

🔴 Agent antwortet nicht:
  1. openclaw gateway status
  2. Logs prüfen: openclaw logs --tail 50
  3. Gateway neustarten: openclaw gateway restart
  4. Wenn weiterhin tot → API-Key prüfen

🟡 Agent antwortet langsam (>30s):
  1. Metriken prüfen: Kontext-Größe, Modell
  2. Memory bereinigen (alte Sessions löschen)
  3. Ggf. auf schnelleres Modell wechseln (Haiku)

🟢 Regelmäßige Wartung:
  1. openclaw doctor (wöchentlich)
  2. Backup prüfen (täglich automatisiert)
  3. Kosten-Report checken (wöchentlich)
  4. Security Audit (monatlich) 🔴 Agent not responding:
  1. openclaw gateway status
  2. Check logs: openclaw logs --tail 50
  3. Restart gateway: openclaw gateway restart
  4. If still dead → check API key

🟡 Agent responding slowly (>30s):
  1. Check metrics: context size, model
  2. Clean up memory (delete old sessions)
  3. Optionally switch to faster model (Haiku)

🟢 Regular maintenance:
  1. openclaw doctor (weekly)
  2. Check backup (daily automated)
  3. Review cost report (weekly)
  4. Security audit (monthly)

Fähigkeit, den internen Zustand eines Systems über seine Outputs (Logs, Metriken, Traces) zu verstehen.Ability to understand the internal state of a system through its outputs (logs, metrics, traces).
Runbook
Dokumentierte Anleitung für typische Fehler-Szenarien und deren Behebung.Documented guide for typical failure scenarios and their resolution.
SRE (Site Reliability Engineering)
Google-Methodik zur Sicherstellung von System-Zuverlässigkeit durch Automatisierung und Monitoring.Google methodology for ensuring system reliability through automation and monitoring.
