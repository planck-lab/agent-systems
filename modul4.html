Tool Use & Kontrollierte AutonomieTool Use & Controlled Autonomy

Wie Agents Werkzeuge nutzen — und wie du kontrollierst, was sie dürfen.How agents use tools — and how you control what they're allowed to do.

🎯 Lernziel (SMART)Learning Goal (SMART)
Innerhalb von 60 min: 2 Tool-Policies konfigurieren und per Log-Analyse nachweisen, dass sie die Autonomie wie erwartet einschränken. Within 60 min: Configure 2 tool policies and prove through log analysis that they restrict autonomy as expected.

❌ Vorher❌ Before

Tool Use ist alles oder nichts. Entweder der Agent darf alles, oder nichts.Tool Use is all or nothing. Either the agent can do everything, or nothing.

✅ Nachher✅ After

Du steuerst granular, welche Tools frei und welche nur mit Bestätigung nutzbar sind.You control granularly which tools are free and which require approval.

MCP — „USB für Agents"MCP — "USB for Agents"

Model Context Protocol (MCP)

Anthropic (2024) definiert MCP als Industriestandard für Tool-Integration. Wie USB für Peripheriegeräte einheitliche Stecker definiert, definiert MCP, wie Agents:Anthropic (2024) defines MCP as the industry standard for tool integration. Like USB defines universal connectors for peripherals, MCP defines how agents:

🔍 Tools entdecken — welche Werkzeuge stehen zur Verfügung?🔍 Discover tools — which tools are available?

📋 Schemas verstehen — welche Parameter braucht ein Tool?📋 Understand schemas — what parameters does a tool need?

⚡ Tools aufrufen — wie wird ein Tool korrekt gestartet?⚡ Call tools — how is a tool correctly invoked?

📚 MCP Specification (Anthropic, 2024): modelcontextprotocol.io Tier C

Tool-Taxonomy: Read-only vs. Side-EffectsTool Taxonomy: Read-only vs. Side Effects

Nicht alle Tools sind gleich gefährlich. Die entscheidende Frage: Verändert das Tool etwas in der Welt?Not all tools are equally risky. The key question: Does the tool change anything in the world?

🟢 Read-only (sicher)🟢 Read-only (safe)

  • Web-SucheWeb search
  • Datei lesenRead file
  • Kalender lesenRead calendar
  • Wetter abfragenCheck weather
  • System-StatusSystem status

🔴 Side-Effects (gefährlich)🔴 Side effects (dangerous)

  • E-Mail sendenSend email
  • Datei schreiben/löschenWrite/delete file
  • Kalender-Termin erstellenCreate calendar event
  • Shell-Befehl ausführenExecute shell command
  • API-Aufruf mit MutationAPI call with mutation

Schick et al. (2023, Toolformer) zeigen: LLMs lernen selbstständig, wann Tools hilfreich sind. Das Problem: Sie unterscheiden nicht zwischen harmlosen und gefährlichen Tools. Policies kanalisieren diese Fähigkeit.Schick et al. (2023, Toolformer) show: LLMs learn on their own when tools are helpful. The problem: They don't distinguish between harmless and dangerous tools. Policies channel this capability.

📚 Schick et al. (2023): "Toolformer: Language Models Can Teach Themselves to Use Tools", NeurIPS 2023 Tier A

Approval Policies: deny → allowlist → fullApproval Policies: deny → allowlist → full

Drei Stufen der Kontrolle — vom Paranoid-Modus bis zum Vertrauensvorschuss:Three levels of control — from paranoid mode to trust-based:

🔒
deny
Alles gesperrtEverything locked
→
📋
allowlist
Nur erlaubte ToolsOnly allowed tools
→
✅
full
Alles erlaubtEverything allowed

Best Practice: Starte immer bei deny und öffne schrittweise. Nicht andersherum. Wie beim Fahren: Erst Schritttempo, dann Autobahn.Best practice: Always start at deny and open up gradually. Not the other way around. Like driving: Walking speed first, then highway.

Beispiel: Zwei Policy-ProfileExample: Two Policy Profiles

🔒 Paranoid-Profil🔒 Paranoid profile

{ "exec": { "security": "deny" }, "tools": { "read": true, "write": false, "web_search": true, "web_fetch": true } }

⚖️ Balanced-Profil⚖️ Balanced profile

{ "exec": { "security": "allowlist", "allow": ["ls", "cat", "grep", "date", "uptime"] }, "tools": { "read": true, "write": true, "web_search": true, "web_fetch": true } }
