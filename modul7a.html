Skills in der Praxis: ClawHubSkills in Practice: ClawHub

Skills sind wiederverwendbare Fähigkeits-Definitionen: Eine SKILL.md-Datei mit Instruktionen + optionale Scripts. Der Agent liest die Datei und weiß, was zu tun ist. ClawHub ist ein Verzeichnis dafür — nutze es als Ideengeber und Referenz, nicht als blinden Installer.Skills are reusable capability definitions: A SKILL.md file with instructions + optional scripts. The agent reads the file and knows what to do. ClawHub is a directory for them — use it as an inspiration and reference, not as a blind installer.

Skill-Bewertung: Checkliste vor der InstallationSkill Assessment: Checklist before installation

Skills von Drittanbietern können beliebigen Code ausführen. Gehe diese Checkliste durch bevor du einen externen Skill nutzt:Third-party skills can execute arbitrary code. Go through this checklist before using an external skill:

✅ Vor Installation prüfenCheck before installing

  1. SKILL.md lesen — Verstehst du was der Skill tut?Read SKILL.md — Do you understand what the skill does?
  2. Scripts prüfen — Welche Shell-Befehle werden ausgeführt?Review scripts — What shell commands are executed?
  3. Berechtigungen — Braucht der Skill write/exec? Warum?Permissions — Does the skill need write/exec? Why?
  4. Autor bekannt? — Wer hat den Skill geschrieben?Known author? — Who wrote the skill?
  5. In Sandbox testen — Erst isoliert ausprobierenTest in sandbox — Try it isolated first

🚫 Red FlagsRed Flags

  • Skill braucht exec: full ohne ErklärungSkill requires exec: full without explanation
  • Obfuskierter oder minifizierter CodeObfuscated or minified code
  • Externe URLs in Scripts (Daten-Exfiltration?)External URLs in scripts (data exfiltration?)
  • Skill überschreibt SOUL.md oder AGENTS.mdSkill overrides SOUL.md or AGENTS.md
  • Kein README, kein Autor, kein ChangelogNo README, no author, no changelog

Empfehlung: Nutze externe Skills als Inspiration und baue deine eigenen (→ Modul 7b). Ein Skill den du verstehst ist sicherer als einer den du nur installierst. Mehr zu Prompt Injection in Skills: → Modul 11: SecurityRecommendation: Use external skills as inspiration and build your own (→ Module 7b). A skill you understand is safer than one you just install. Skill-Routing in AGENTS.mdSkill Routing in AGENTS.md

Der Agent braucht eine Routing-Tabelle: Welche Absicht → Welcher Skill? Das definierst du in AGENTS.md:The agent needs a routing table: Which intent → Which skill? You define this in AGENTS.md:

### Skill-Routing (automatisch) Wenn eine Nachricht zu einem Skill passt, lies die Instruction-Datei und folge ihr — ohne Rückfrage. | Absicht | Skill | |----------------------------|------------------| | "Recherchiere", "/research"| deep-research/ | | "Qualitätscheck", "/qcheck"| quality-gate/ | | "LinkedIn Post", "Content" | create-content/ | | "Bewerte Quelle" | quellen-check/ | | "Paper zu...", "arXiv" | paper-reader/ |### Skill Routing (automatic) When a message matches a skill, read the instruction file and follow it — without asking. | Intent | Skill | |-----------------------------|------------------| | "Research", "/research" | deep-research/ | | "Quality check", "/qcheck" | quality-gate/ | | "LinkedIn post", "Content" | create-content/ | | "Evaluate source" | source-check/ | | "Paper about...", "arXiv" | paper-reader/ |

Prinzip: Absicht erkennen → Datei lesen → Schritten folgen. Nicht fragen, einfach machen.Principle: Recognize intent → read file → follow steps. Don't ask, just do it.
