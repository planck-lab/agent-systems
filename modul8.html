Die 5 Agent-FehlermodiThe 5 Agent Failure Modes

Jeder Agent wird irgendwann scheitern. Der Unterschied: Du erkennst das Muster und weißt, was zu tun ist.Every agent will fail at some point. The difference: You recognize the pattern and know what to do.

🌫️
HalluzinationHallucination
Agent erfindet Fakten, Dateien oder API-Antworten, die nicht existieren.Agent invents facts, files, or API responses that don't exist.
Fix: Quellen-Verifizierung, Tool-Output prüfenFix: Source verification, check tool output
🔧
Tool-Misuse
Agent ruft falsches Tool auf oder übergibt falsche Parameter. Z.B. rm statt ls.Agent calls wrong tool or passes wrong parameters. E.g., rm instead of ls.
Fix: Allowlists, Approval Policies (→ M4)Fix: Allowlists, approval policies (→ M4)
🔄
Loop / Recursion
Agent wiederholt dieselbe Aktion endlos. Kosten steigen, kein Fortschritt.Agent repeats the same action endlessly. Costs rise, no progress.
Fix: Max-Iterations, Loop-DetectionFix: Max iterations, loop detection
💥
Context-Overflow
Kontext-Fenster voll → Agent „vergisst" Instruktionen oder frühe Nachrichten.Context window full → agent "forgets" instructions or early messages.
Fix: Memory-Hierarchie (→ M6), Kontext-ManagementFix: Memory hierarchy (→ M6), context management
⚠️
Unbeabsichtigte AktionenUnintended Actions
Agent interpretiert Aufgabe korrekt, führt aber unerwünschte Nebenwirkungen aus. Z.B. löscht Dateien beim „Aufräumen".Agent interprets task correctly but executes unintended side effects. E.g., deletes files when "cleaning up".
Fix: Side-Effect-Policies, HITL für destruktive AktionenFix: Side-effect policies, HITL for destructive actions

(2023) zeigen einen weiteren kritischen Fehlermodus: Indirekte Prompt Injection — Angreifer schleusen Instruktionen über externe Inhalte (Webseiten, E-Mails) ein, die der Agent ausführt, ohne dass der Nutzer es merkt. Dies verbindet Halluzination und Tool-Misuse zu einer Security-Bedrohung.Greshake et al. (2023) demonstrate another critical failure mode: Indirect Prompt Injection — attackers inject instructions via external content (web pages, emails) that the agent executes without the user noticing. This combines hallucination and tool-misuse into a security threat.

Ethik: Wann SOLL ein Agent NICHT handeln?Ethics: When Should an Agent NOT Act?

Die wichtigste Fähigkeit eines Agents ist nicht Handeln — sondern zu wissen, wann er aufhören muss.The most important skill of an agent is not acting — but knowing when to stop.

  • 🚫 Unsichere AktionenUnsafe Actions: Destruktive Operationen (Löschen, Senden, Publizieren) ohne explizite FreigabeDestructive operations (delete, send, publish) without explicit approval
  • 🧠 Bias & VorurteilePrejudice: Agent darf keine Entscheidungen treffen, die Menschen benachteiligen (Bewerbungsfilter, Scoring)Agent must not make decisions that disadvantage people (application filters, scoring)
  • 🔒 Privacy: Persönliche Daten nicht ohne Consent verarbeiten. Keine Arbeitskontext-Daten mischen.Don't process personal data without consent. Don't mix work-context data.
  • 📢 TransparenzTransparency: Agent soll sagen, dass er ein Agent ist. Keine Mimikry, kein Vortäuschen.Agent should state that it is an agent. No mimicry, no pretending.
  • ⚖️ VerantwortlichkeitAccountability: Mensch bleibt verantwortlich. Agent liefert Empfehlung, Mensch entscheidet.Human remains responsible. Agent provides recommendation, human decides.
