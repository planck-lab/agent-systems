Schichten verstehen — vom Telegram-Chat bis zur Tool-Ausführung.Understanding layers — from Telegram chat to tool execution.

🎯 Lernziel (SMART)Learning Goal (SMART)
Innerhalb von 60 min: Die 4 Schichten eines Orchestrators auf einem Diagramm zuordnen und je eine Konfiguration vornehmen. Within 60 min: Map the 4 layers of an orchestrator on a diagram and configure one setting for each.

❌ Vorher❌ Before

✅ Nachher✅ After

Du verstehst Channel → Gateway → Session → Runtime als unabhängige Schichten.You understand Channel → Gateway → Session → Runtime as independent layers.

Die 4 SchichtenThe 4 Layers

Jedes Agent-System — egal ob LangGraph, CrewAI oder OpenClaw — hat diese grundlegenden Schichten. Die Namen unterscheiden sich, das Prinzip bleibt gleich:Every agent system — whether LangGraph, CrewAI, or OpenClaw — has these fundamental layers. Names differ, but the principle stays the same:

┌─────────────────────────────────────────────────────┐ │ Schicht 1: CHANNELS (Input/Output) │ │ Telegram · WhatsApp · Web-UI · CLI · API │ │ → Abstrahiert die Plattform │ ├─────────────────────────────────────────────────────┤ │ Schicht 2: GATEWAY (Control Plane) │ │ Routing · Auth · Rate-Limiting · DM-Policy │ │ → Entscheidet WER mit WELCHEM Agent spricht │ ├─────────────────────────────────────────────────────┤ │ Schicht 3: SESSIONS (State Management) │ │ Kontext-History · Memory · Isolation │ │ → Jede Konversation ist isoliert │ ├─────────────────────────────────────────────────────┤ │ Schicht 4: RUNTIME (Execution) │ │ LLM · Tools · Skills · Approval-Checks │ │ → Hier passiert die eigentliche Arbeit │ └─────────────────────────────────────────────────────┘┌─────────────────────────────────────────────────────┐ │ Layer 1: CHANNELS (Input/Output) │ │ Telegram · WhatsApp · Web UI · CLI · API │ │ → Abstracts the platform │ ├─────────────────────────────────────────────────────┤ │ Layer 2: GATEWAY (Control Plane) │ │ Routing · Auth · Rate limiting · DM policy │ │ → Decides WHO talks to WHICH agent │ ├─────────────────────────────────────────────────────┤ │ Layer 3: SESSIONS (State Management) │ │ Context history · Memory · Isolation │ │ → Each conversation is isolated │ ├─────────────────────────────────────────────────────┤ │ Layer 4: RUNTIME (Execution) │ │ LLM · Tools · Skills · Approval checks │ │ → This is where the actual work happens │ └─────────────────────────────────────────────────────┘
1
Channels — Die EingangstürChannels — The Front Door
Verschiedene Plattformen → ein Interface. Ob Telegram, WhatsApp oder CLI — der Agent bekommt immer dasselbe Format. Konfiguration: Welche Channels aktiviert? Welche Rechte?Different platforms → one interface. Whether Telegram, WhatsApp, or CLI — the agent always gets the same format. Config: Which channels enabled? What permissions?
2
Gateway — Der TürsteherGateway — The Bouncer
Routing, Authentifizierung, Rate-Limiting. dmPolicy: "pairing" bedeutet: Nur gepaarte Nutzer dürfen kommunizieren. Konfiguration: Wer darf rein? Wie viele Anfragen pro Minute?Routing, authentication, rate limiting. dmPolicy: "pairing" means: Only paired users can communicate. Config: Who gets in? How many requests per minute?
3
Sessions — Das GedächtnisSessions — The Memory
Jede Konversation ist isoliert — Session A sieht nichts von Session B. Context-History, Memory-Dateien, Session-Timer. Konfiguration: Wie lang bleibt eine Session aktiv? Was wird gespeichert?Each conversation is isolated — Session A can't see Session B. Context history, memory files, session timer. Config: How long does a session stay active? What gets saved?
4
Runtime — Die WerkstattRuntime — The Workshop
LLM-Aufruf, Tool-Execution, Skill-Routing, Approval-Checks. Hier passiert die eigentliche Arbeit. Konfiguration: Welches Modell? Welche Tools erlaubt?LLM call, tool execution, skill routing, approval checks. This is where the actual work happens. Config: Which model? Which tools allowed?

📖 Begriffe in diesem Modul📖 Terms in this module

Gateway
Vermittlungsschicht zwischen Channels und Agent-Runtime — der Türsteher.Mediation layer between channels and agent runtime — the bouncer.
Session
Isolierter Gesprächskontext mit eigener History und eigenem State.Isolated conversation context with its own history and state.
Channel
Plattform-Anbindung (Telegram, WhatsApp, CLI) — die Eingangstür.Platform connection (Telegram, WhatsApp, CLI) — the front door.
Runtime
Die Ausführungsschicht: LLM-Aufruf, Tool-Execution, Approval-Checks.The execution layer: LLM call, tool execution, approval checks.
