SKILL.md AnatomieSKILL.md Anatomy

Jeder Skill besteht aus einer SKILL.md-Datei. Diese hat zwei Teile: YAML-Frontmatter (Metadaten) und Instructions (was der Agent tun soll).Every skill consists of a SKILL.md file. This has two parts: YAML frontmatter (metadata) and instructions (what the agent should do).

--- name: wetter-briefing version: 1.0.0 description: Tägliches Wetter-Briefing mit Empfehlung author: dein-name triggers: - "Wetter" - "Wie wird das Wetter?" - "/wetter" tools_required: - web_search - web_fetch tags: [wetter, daily, briefing] --- # Wetter-Briefing ## Ablauf 1. Lies die Zeitzone aus USER.md 2. Suche nach dem aktuellen Wetter am Standort des Users 3. Fasse zusammen: Temperatur, Bewölkung, Niederschlag 4. Gib eine Empfehlung (z.B. "Regenschirm mitnehmen") 5. Halte die Antwort unter 3 Sätzen ## Format - Emoji am Anfang: ☀️🌤️🌧️❄️ - Temperatur in °C - Empfehlung kursiv ## Beispiel-Output ☀️ Berlin: 18°C, sonnig, leichte Brise. *Perfektes Jogging-Wetter — ab nach draußen!*--- name: weather-briefing version: 1.0.0 description: Daily weather briefing with recommendation author: your-name triggers: - "Weather" - "What's the weather like?" - "/weather" tools_required: - web_search - web_fetch tags: [weather, daily, briefing] --- # Weather Briefing ## Steps 1. Read timezone from USER.md 2. Search for current weather at user's location 3. Summarize: temperature, cloud cover, precipitation 4. Give a recommendation (e.g. "Take an umbrella") 5. Keep the answer under 3 sentences ## Format - Emoji at the start: ☀️🌤️🌧️❄️ - Temperature in °C - Recommendation in italics ## Example Output ☀️ Berlin: 18°C, sunny, light breeze. *Perfect jogging weather — get outside!*

📋 YAML-Frontmatter (Pflichtfelder)📋 YAML Frontmatter (Required Fields)

name — Eindeutiger Skill-Name · version — Semantic Versioning · description — Was der Skill macht · triggers — Wann wird er aktiviert? · tools_required — Welche Tools braucht er?name — Unique skill name · version — Semantic versioning · description — What the skill does · triggers — When is it activated? · tools_required — Which tools does it need?

📝 Instructions (der Kern)📝 Instructions (the core)

Klar formulierte Schritte, die der Agent befolgt. Format: Markdown mit nummerierten Schritten. Je präziser, desto konsistenter das Ergebnis. Tipp: Immer ein Beispiel-Output mitliefern — das verbessert die Qualität erheblich.Clearly formulated steps for the agent to follow. Format: Markdown with numbered steps. The more precise, the more consistent the result. Tip: Always include example output — this significantly improves quality.

Shell-Script erstellenCreating a Shell Script

Manche Skills brauchen ein Script für Aufgaben, die der Agent nicht direkt kann (z.B. API-Aufrufe mit speziellem Format, Datei-Verarbeitung):Some skills need a script for tasks the agent can't do directly (e.g. API calls with special format, file processing):

#!/bin/bash # wetter.sh — Wetter-Abfrage mit formatierter Ausgabe # Usage: wetter.sh [STADT] CITY="${1:-Berlin}" # API-Aufruf (Beispiel mit wttr.in) RESULT=$(curl -s "wttr.in/${CITY}?format=3" 2>/dev/null) if [ -z "$RESULT" ]; then echo "❌ Wetter-Abfrage fehlgeschlagen für: $CITY" exit 1 fi echo "$RESULT"#!/bin/bash # weather.sh — Weather query with formatted output # Usage: weather.sh [CITY] CITY="${1:-Berlin}" # API call (example with wttr.in) RESULT=$(curl -s "wttr.in/${CITY}?format=3" 2>/dev/null) if [ -z "$RESULT" ]; then echo "❌ Weather query failed for: $CITY" exit 1 fi echo "$RESULT"

Wichtig: chmod +x wetter.sh nicht vergessen! Und immer Error-Handling einbauen.Important: Don't forget chmod +x weather.sh! And always include error handling.

Skill bauen: Schritt für SchrittBuilding a Skill: Step by Step

  1. Idee formulieren: Was soll der Skill können? Welche Trigger? Welche Tools?Formulate idea: What should the skill do? Which triggers? Which tools?
  2. SKILL.md schreiben: YAML-Frontmatter + klare Schritt-für-Schritt-Instruktionen + Beispiel-OutputWrite SKILL.md: YAML frontmatter + clear step-by-step instructions + example output
  3. Shell-Script erstellen (falls nötig): Error-Handling, Exit-Codes, chmod +xCreate shell script (if needed): Error handling, exit codes, chmod +x
  4. Routing ergänzen: Trigger in AGENTS.md Skill-Routing-Tabelle eintragenAdd routing: Enter triggers in AGENTS.md skill routing table
  5. Testen: 5 verschiedene Formulierungen → Wird der Skill korrekt getriggert?Test: 5 different phrasings → Is the skill correctly triggered?
  6. Publizieren (optional): README.md schreiben, auf ClawHub veröffentlichenPublish (optional): Write README.md, publish to ClawHub

Warum M7a (nutzen) und M7b (bauen) getrennt? Bloom's Taxonomy: „Anwenden" (M7a) vs. „Erstellen" (M7b) sind zwei kognitive Stufen auseinander. Cognitive Load Theory (Sweller, 2019) argumentiert: Beide in einem Modul würde Power-User überfordern und Developer ablenken. Die Trennung respektiert das.Why M7a (using) and M7b (building) separate? Bloom's Taxonomy: "Apply" (M7a) vs. "Create" (M7b) are two cognitive levels apart. Cognitive Load Theory (Sweller, 2019) argues: Both in one module would overwhelm power users and distract developers. The separation respects this.
