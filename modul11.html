Trust Model: Personal Assistant SecurityTrust Model: Personal Assistant Security

Ein persönlicher Agent hat ein einzigartiges Security-Profil: Er hat Zugriff auf deine Dateien, Nachrichten, APIs — und handelt in deinem Namen. Das erfordert ein Trust Model, das diese drei Schichten absichert:A personal agent has a unique security profile: it has access to your files, messages, APIs — and acts on your behalf. This requires a trust model that secures these three layers:

BedrohungsvektorenThreat Vectors

💉
Direkte Prompt InjectionDirect Prompt Injection
Nutzer gibt manipulierten Prompt ein, um System-Instruktionen zu überschreiben.User enters manipulated prompt to override system instructions.
Defense: Input-Sanitization, feste System-Prompt-GrenzenDefense: Input sanitization, fixed system prompt boundaries
🕸️
Indirekte Prompt InjectionIndirect Prompt Injection
Externe Inhalte (Webseiten, E-Mails) enthalten versteckte Instruktionen, die der Agent ausführt.External content (web pages, emails) contains hidden instructions that the agent executes.
Defense: Content-Isolation, Output-Validierung (Greshake et al.)Defense: Content isolation, output validation (Greshake et al.)
📂
Daten-ExfiltrationData Exfiltration
Agent leitet sensible Daten (API-Keys, persönliche Infos) nach außen weiter.Agent leaks sensitive data (API keys, personal info) to external parties.
Defense: Gesperrte Pfade, Output-Filter, AllowlistsDefense: Blocked paths, output filters, allowlists
🔄
Privilege EscalationPrivilege Escalation
Agent versucht, seine Berechtigungen zu erweitern (z.B. Shell-Escape, Policy-Änderung).Agent attempts to expand its permissions (e.g., shell escape, policy change).
Defense: Sandboxing, immutable Policies, Approval für ÄnderungenDefense: Sandboxing, immutable policies, approval for changes

AI Control: Sicherheit auch bei intentionaler SubversionAI Control: Safety Even Under Intentional Subversion

Greenblatt et al. (2024) argumentieren radikal: Selbst wenn ein Agent absichtlich gegen seine Instruktionen arbeitet (z.B. durch Training-Artefakte oder Manipulation), müssen Kontrollmechanismen greifen. Zwei Kernstrategien:Greenblatt et al. (2024) argue radically: Even if an agent intentionally works against its instructions (e.g., through training artifacts or manipulation), control mechanisms must hold. Two core strategies:

🔍 Monitoring: Alle Aktionen loggen und auf Anomalien prüfen. Nicht vertrauen — verifizieren.Log all actions and check for anomalies. Don't trust — verify.

⏸️ Pausing: Bei Verdacht sofort pausieren. Lieber eine valide Aktion blockieren als eine schädliche durchlassen.Pause immediately on suspicion. Better to block a valid action than to let a harmful one through.

📚 Greenblatt et al. (2024): "AI Control: Improving Safety Despite Intentional Subversion", arXiv:2312.06942 Tier B

📖 Begriffe in diesem Modul📖 Terms in this module

Prompt Injection
Angriff, bei dem Nutzer-Eingaben oder externe Inhalte die Agent-Instruktionen manipulieren (direkt oder indirekt).Attack where user input or external content manipulates agent instructions (direct or indirect).
Threat Model
Strukturierte Dokumentation von Assets, Bedrohungen, Auswirkungen und Gegenmaßnahmen.Structured documentation of assets, threats, impacts, and countermeasures.
Daten-ExfiltrationData Exfiltration
Ungewollte Weitergabe sensibler Daten an externe Systeme.Unwanted transfer of sensitive data to external systems.
Sandboxing
Isolierte Ausführungsumgebung, die den Agent-Zugriff auf Systemressourcen begrenzt.Isolated execution environment that limits agent access to system resources.
