Autonomie-Budget: Automatisierung braucht engere GrenzenAutonomy Budget: Automation Needs Tighter Limits

Wenn ein Agent ohne menschliche Aufsicht handelt (Cron, Webhook), braucht er striktere Policies als im interaktiven Chat. Das Autonomie-Budget definiert: Wie viel darf der Agent allein entscheiden?When an agent acts without human oversight (cron, webhook), it needs stricter policies than in interactive chat. The autonomy budget defines: How much can the agent decide on its own?

Niedrig: Nur lesenLow: Read only
Agent darf nur Informationen sammeln und berichten. Keine Aktionen.Agent may only collect information and report. No actions.
Mittel: Lesen + eingeschränkt handelnMedium: Read + limited actions
Agent darf sammeln UND vordefinierte Aktionen ausführen (z.B. Nachricht senden). Keine destruktiven Ops.Agent may collect AND execute predefined actions (e.g., send message). No destructive ops.
Hoch: Volle Autonomie (gefährlich!)High: Full autonomy (dangerous!)
Agent darf alles — nur sinnvoll mit robusten Safety-Checks und Monitoring (→ M11, M13).Agent can do anything — only sensible with robust safety checks and monitoring (→ M11, M13).

HITL-Eskalation implementierenImplementing HITL Escalation

Der wichtigste Teil jeder Automatisierung: Wann eskaliert der Agent an den Menschen?The most important part of any automation: When does the agent escalate to the human?

Eskalations-Regeln (Beispiel)Escalation Rules (Example)

1. Kosten-Threshold: Wenn Task > $5 kostet → Eskalation
2. Unsicherheit: Wenn Agent keine Lösung findet nach 3 Versuchen → Eskalation
3. Destruktiv: Jede Lösch-/Sende-Aktion in Cron-Jobs → IMMER Eskalation
4. Anomalie: Wenn Ergebnis stark von Erwartung abweicht → Eskalation 1. Cost threshold: If task costs > $5 → escalate
2. Uncertainty: If agent can't find solution after 3 attempts → escalate
3. Destructive: Any delete/send action in cron jobs → ALWAYS escalate
4. Anomaly: If result differs significantly from expectation → escalate
