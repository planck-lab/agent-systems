Orchestration Patterns im DetailOrchestration Patterns in Detail

1Sequential (Pipeline)
Agent A → Agent B → Agent C. Jeder verarbeitet den Output des Vorgängers. Einfach, deterministisch, gut debugbar.Agent A → Agent B → Agent C. Each processes the predecessor's output. Simple, deterministic, easy to debug.
Sequential Pipeline: ┌───────────────┐ ┌───────────────┐ ┌───────────────┐ │ Researcher │───▶│ Summarizer │───▶│ Formatter │ │ (Web Search) │ │ (Condensing) │ │ (Markdown) │ └───────────────┘ └───────────────┘ └───────────────┘ Input Output
2Concurrent (Fan-Out / Fan-In)
Mehrere Agents parallel → Ergebnisse zusammenführen. Schneller, aber komplexere Error-Handling.Multiple agents in parallel → merge results. Faster, but more complex error handling.
Concurrent Fan-Out / Fan-In: ┌──────────────────┐ ┌───▶│ Agent A: arxiv │───┐ │ └──────────────────┘ │ ┌─────────┐ │ ┌──────────────────┐ │ ┌───────────┐ │ Main │──┼───▶│ Agent B: web │───┼───▶│ Merge & │ │ Agent │ │ └──────────────────┘ │ │ Synthesize│ └─────────┘ │ ┌──────────────────┐ │ └───────────┘ └───▶│ Agent C: news │───┘ └──────────────────┘
3Handoff
Agent A erkennt: "Das kann ich nicht" → delegiert mit vollem Kontext an Agent B. Intelligent routing basierend auf Kompetenz.Agent A recognizes: "I can't do this" → delegates with full context to Agent B. Intelligent routing based on competence.
4Orchestrator-Worker
Ein Manager-Agent plant, verteilt und konsolidiert. Worker-Agents führen aus. Am mächtigsten, aber am komplexesten.A manager agent plans, distributes, and consolidates. Worker agents execute. Most powerful, but most complex.

Aufgabe: "Recherchiere AI Agents in Healthcare"

1. Decompose:
  → Sub-Task A: "Aktuelle Papers auf arXiv zu AI Agents Healthcare"
  → Sub-Task B: "Industriebeispiele und Startups"
  → Sub-Task C: "Regulatorische Herausforderungen (FDA, EU AI Act)"

2. Spawn:
  sessions_spawn(task: Sub-Task A, model: "sonnet", timeout: 180)
  sessions_spawn(task: Sub-Task B, model: "haiku", timeout: 120)
  sessions_spawn(task: Sub-Task C, model: "sonnet", timeout: 180)

3. Synthesize:
  → Ergebnisse zusammenführen, Duplikate entfernen, Briefing erstellen Task: "Research AI Agents in Healthcare"

1. Decompose:
  → Sub-Task A: "Recent papers on arXiv about AI Agents Healthcare"
  → Sub-Task B: "Industry examples and startups"
  → Sub-Task C: "Regulatory challenges (FDA, EU AI Act)"

2. Spawn:
  sessions_spawn(task: Sub-Task A, model: "sonnet", timeout: 180)

3. Synthesize:
  → Merge results, remove duplicates, create briefing

Error Handling: Wenn Sub-Agents scheiternError Handling: When Sub-Agents Fail

🔄 Retry: Sub-Agent nochmal starten (max 2x). Sinnvoll bei Timeouts.Restart sub-agent (max 2x). Makes sense for timeouts.

⏭️ Skip: Teilaufgabe überspringen, Ergebnis ohne diesen Teil zusammenbauen.Skip sub-task, build result without this part.

🚨 Escalate: An den Menschen eskalieren: "Sub-Agent C gescheitert — hier das Teilergebnis."Escalate to human: "Sub-agent C failed — here's the partial result."
